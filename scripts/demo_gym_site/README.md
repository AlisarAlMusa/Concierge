# IronFit Gym — Concierge widget demo site

A fully static, single-folder gym website that embeds the **Concierge AI
Assistant** widget. Pretend you're a website owner who pasted the embed
snippet generated by the Admin panel — that's exactly what this site is.

## Run

```bash
cd scripts/demo_gym_site
python3 -m http.server 5500
```

Then open **http://localhost:5500/index.html**.

The chat bubble appears in the lower-right corner. Click it, ask anything
about IronFit, and the assistant answers from the published CMS pages.

## Architecture

```
Browser (http://localhost:5500)
   │
   ├── GET *.html, *.css, *.js          ← served by python -m http.server
   │
   └── <script src="http://localhost:8000/widget.js"
                data-widget-id="pub_wid_ironfit_001" async>
            │
            ├── POST  http://localhost:8000/public/widgets/session
            ├── GET   http://localhost:8000/public/widgets/config
            └── POST  http://localhost:8000/public/chat
```

CORS is handled by the API's `WidgetCorsMiddleware`; the widget's
`allowed_origins` list (set in `provision_ironfit.py`) gates session
minting server-side.

## Provisioning

The provisioning script creates the tenant, admin user, widget, and 6
published-and-indexed CMS pages. It is **idempotent** — safe to re-run.

```bash
# From the host:
docker cp Concierge/scripts/demo_gym_site/provision_ironfit.py concierge-api-1:/tmp/
docker exec -e CONCIERGE_API=http://localhost:8000 concierge-api-1 \
    python /tmp/provision_ironfit.py
```

Resulting credentials:

| What | Value |
|---|---|
| Tenant | `IronFit Gym` (slug `ironfit-gym`) |
| Tenant admin | `admin@ironfit-gym.com` / `Admin!2026` |
| Public widget id | `pub_wid_ironfit_001` |
| Allowed origins | `http://localhost:5500`, `http://127.0.0.1:5500` |
