## Implementation Phases

### Phase A1 — Fix pytest bypass in ci.yml

Remove `|| true` from the `pytest` step. Tests must fail CI. Add `continue-on-error: false` (default, but be explicit).

**Files**: `.github/workflows/ci.yml`

**Acceptance**: A deliberately broken test in a branch causes the `lint-and-test` job to fail.

---

### Phase A2 — Upgrade uv setup in ci.yml

Replace `pip install uv` + `uv pip install --system -e ".[dev]"` with the official `astral-sh/setup-uv@v5` action and `uv sync --group dev`.

**Files**: `.github/workflows/ci.yml`

**Acceptance**: CI still passes after the change. uv binary is cached between runs.

---

### Phase A3 — Extend smoke test to cover sidecar health endpoints

Add `GET http://localhost:8001/health` (model_server) and `GET http://localhost:8002/health` (guardrails_sidecar) to the smoke-test CI job and to `scripts/smoke_test.sh`.

**Files**: `.github/workflows/ci.yml`, `scripts/smoke_test.sh`

**Acceptance**: Smoke test fails if any of the three services' `/health` endpoints return non-200.

---

### Phase A4 — Wire eval-gates CI job

Uncomment and fully wire the `eval-gates` job. One named step per eval gate. Each step calls the corresponding eval script. The job has `needs: smoke-test`. The eval scripts' exit codes control the step outcome.

**Files**: `.github/workflows/ci.yml`

**Acceptance**: The eval-gates job appears in the GitHub Actions workflow graph. Each gate is a separately named step. A non-zero exit from any script fails that step and blocks merge.

---

### Phase A5 — Scaffold run_evals.sh with dispatch + exit-code semantics

Replace the TODO stub with a real dispatcher that:
- Accepts `classifier | rag | agent | security | redaction | all` as argument.
- Checks that `evals/eval_thresholds.yaml` exists; exits 1 with a clear message if not.
- For each suite, checks that the corresponding golden set / test file exists; exits 1 if not.
- Delegates to `uv run python evals/<suite>/evaluate_<suite>.py --thresholds evals/eval_thresholds.yaml`.
- Propagates the exit code from the delegate script.
- Runs all suites in sequence for `all`, accumulating failures and exiting 1 if any failed.

**Files**: `scripts/run_evals.sh`

**Acceptance**: `bash scripts/run_evals.sh classifier` exits 1 with "golden set not found" when `evals/classifier/test_set.csv` is absent. Once Person C adds the scripts, the same call exits 0 or 1 based on eval result.
